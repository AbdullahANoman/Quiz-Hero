Question-1 : Difference Between forEach, map, filter and find ?

Answer :

ForEach, Map, Filter, and Find are all array methods in JavaScript, but they differ in their functionality and the way they modify or return arrays.

forEach :

forEach iterates over each element in an array and performs a function on each element, but it does not modify the original array. The return value of forEach is always undefined.


map :

map also iterates over each element in an array and performs a function on each element, but it creates a new array with the results of that function applied to each element. The original array remains unchanged.


filter :

filter creates a new array with all elements that pass a test specified by a function. The original array remains unchanged.


find :

find returns the first element in an array that passes a test specified by a function. If no element passes the test, undefined is returned.

Question-2 : Difference Between Local Storage , Session Storage and Cookies ?

Answer :

localStorage, sessionStorage, and cookies are all used for storage in web browsers, but they differ in their functionality and limitations.

LocalStorage :

localStorage will persist even after the user closes the browser or restarts their computer. localStorage is shared between all tabs and windows from the same origin. This storage store up to 5-10 mb of data,depending on the browser.


SessionStorage :

It is same to local storage in terms of syntax and usage .but it has a difference that the data is only available for the duration of the browser session.


Cookies :

Cookies are small text files that are stored on the user's computer by the web server. They are often used for session management, user preferences, and tracking. Cookies can have an expiration date and can be set to persist across sessions.

Question-3 : What is event loop and how it was work ?

Answer :

Event Loop :

JavaScript has a runtime model based on an event loop, which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks.JS executes all operations on a single thread, but using a few smart data structures, it gives us the illusion of multi-threading.


Here is how the event loop works :

  1. 1.The event loop has an event queue where it stores all the events that need to be processed.
  2. 2.When an event occurs (such as a user clicking a button), it is added to the event queue
  3. 3.The event loop continuously checks the event queue for any pending events. If there is an event in the queue, the event loop removes it from the queue and processes it.
  4. 4.The event loop assigns the event to the appropriate callback function, which is then executed.
  5. 5.While the callback function is being executed, the event loop continues to check the event queue for any pending events.
  6. 6.If there are no more events in the queue, the event loop waits for new events to be added to the queue.
  7. 7. Once the callback function is completed, any new events that were generated during the execution of the callback function are added to the event queue, and the event loop starts the process over again.

Answer:
